# Mermaid 图自检

画完 Mermaid 图别直接交付，先过一遍这个检查清单。图渲染失败或暗色背景下看不清，比没有图更糟——用户得自己 debug 你的图。

## 为什么

Mermaid 对语法很敏感：节点文本里的特殊字符、括号、引号没转义就会整张图渲染失败。而暗黑主题下，默认浅色描边和深色背景对比不足，箭头和文字容易糊成一团。这两点是画图最常翻车的地方。

## 语法自检清单

逐条核对，有问题就改：

1. **节点文本含特殊字符要用引号包裹**。文本里有 `()`、`[]`、`{}`、`:`、`,`、`"`、`/` 等，必须写成 `A["文本(带括号)"]`，否则解析失败。
2. **不要在节点 ID 里用中文或空格**。ID 用 `A`、`B1`、`node1` 这类，中文放进引号文本里。
3. **箭头语法正确**。flowchart 用 `-->`、`-.->`、`==>`；时序图用 `->>`、`-->>`。别混用。
4. **每个 subgraph 要有 end**。漏掉 `end` 是常见错误。
5. **条件分支标签用 `|文本|`**：`A -->|是| B`，标签里有特殊字符同样要小心。
6. **换行用 `<br/>`** 而不是真实换行。

## 暗黑主题可读性

项目主题是暗色（见 `design-tokens.css`）。确保图在深色背景上清晰：

- 优先让图自带足够对比，不依赖浅色默认描边。需要时用 `init` 指令显式设主题：

```
%%{init: {'theme':'dark'}}%%
```

- 关键节点用 `style` 加填充色和对比文字色，避免深底深字：

```
style A fill:#2563eb,stroke:#60a5fa,color:#fff
```

- 不要用纯黑/极深色作为节点填充（会和背景融为一体），也不要用纯黄等高亮色作大面积填充（刺眼）。

## 验证方法

光看不够，实际渲染一遍最可靠。在 sandbox 里用 mermaid CLI 验证语法：

```bash
# 把图写进 test.mmd，然后
npx -y @mermaid-js/mermaid-cli -i test.mmd -o test.svg 2>&1
```

能成功输出 SVG 说明语法没问题。报错就按错误信息定位行号修复。如果环境装不了 CLI，至少逐条对照上面的语法清单人工核对一遍。

## 完成标准

- 语法清单逐条通过
- 节点特殊字符已转义/加引号
- 暗色背景下对比清晰（关键节点已设 style 或 theme）
- 条件允许时已用 CLI 实际渲染验证
